BOSS - Better Oblivion Sorting Software (and BOSS-F)
Build Instructions

This guide presumes that you are building on Windows. If not, presumably you are experienced enough to know what you're doing. If after following this guide, you are still having trouble compiling BOSS, feel free to PM pacificmorrowind or WrinklyNinja for help.

Preparing Your Environment/Getting Prerequisites

Download and install:

  • Microsoft Visual C++ 2010 Express Edition (if you've already got Visual Studio, just use that)
  • BOOST C++ libraries
  • An SVN client of your choice.
If you're wanting to build BOSS, and not just BOSS-F, you will also need:
  • Windows SDK - You only need to install the "x86 Libraries" and "Windows Headers" from the "Windows Headers And Libraries" section.
  • Curl Source Code - Download the latest zipped source code from the top of the page.

Preparing BOSS and BOOST

  1. Perform a SVN Checkout from https://better-oblivion-sorting-software.googlecode.com/svn to a local directory of your own choice to create your working copy. (This directory shall be referred to as $BOSS from here on in.)
  2. Extract the BOOST libaries archive you downloaded to $BOSS\BOSS source code\$BOOST_ROOT where $BOOST_ROOT is the root of the archive extracted.
  3. Start a Command Prompt and cd to $BOSS\BOSS source code\$BOOST_ROOT and run bootstrap.bat
  4. Edit the file project-config.jam in $BOSS\BOSS source code\$BOOST_ROOT to refer to your version of Visual Studio - for Visual Studio 2010 this is using msvc : 10.0 ; . (For Visual Studio 2008, it would be using msvc : 9.0 ; )
  5. In the Command Prompt, run bjam.exe --build-type=complete. This will take a while. Once that is done, you are ready to build BOSS.

Preparing Curl dependency (not required for building BOSS-F)

We are going to be building the curl requirements from source, since the Windows curl packages are often not up-to-date. We don't need to compile all of curl though, just the static library. To do so, follow these steps:

  1. Extract the curl archive to $BOSS\BOSS source code\curl (there should be the README file, etc in this directory, not a curl-x.xx.x folder).
  2. Open the libcurl.vcproj file found in $BOSS\BOSS source code\curl\lib in Visual C++ 2010.
  3. Set the build type to Release. In Project->Properties->Configuration Properties->C/C++->Code Generation, set the Runtime Library to "Multi-threaded (/MT)".
  4. Build the libcurl project. This will create a libcurl.lib file in the lib/Release directory.


Building BOSS

Open $BOSS\BOSS source code\BOSS.sln in Visual C++ 2010, then select the desired build type (Release or Debug) and run Build on whichever project(s) you are interested in, or just run Build All. The generated exe(s) will be located in $BOSS\BOSS source code\%BUILDTYPE%\
