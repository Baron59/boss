BOSS - Better Oblivion Sorting Software: Build Instructions

This guide presumes that you are building on Windows. If not, presumably you are experienced enough to know what you're doing. If after following this guide, you are still having trouble compiling BOSS, feel free to PM WrinklyNinja for help.

This guide also assumes that you are building the 'trunk' of the boss-common code. If you are building a branch, change all references to trunk\ to the directory that contains the BOSS.sln file of the code you are building.


Preparing Your Environment/Getting Prerequisites

Download and install:

  • Microsoft Visual C++ 2010 Express Edition
  • Microsoft Visual C++ 2008 Express Edition (required for compatibility with Windows 2000)
  • Curl Source Code - Download the latest zipped source code from the top of the page.
  • BOOST C++ libraries - BOSS requires version 1.46.1, it will not work with earlier versions, later versions can be tried, but might also break something.
  • UTF8-CPP - Download the latest version.
  • An SVN client of your choice.

Preparing BOSS, UTF8-CPP and BOOST

  1. Perform a SVN Checkout from https://better-oblivion-sorting-software.googlecode.com/svn to a local directory of your own choice to create your working copy. (This directory shall be referred to as $BOSS from here on in.)
  2. Extract the contents of the utf8_vx_x_x (x is a number) archive to $BOSS\lib\utf8\, so that your utf8 folder contains a 'doc' and 'source' folder.
  3. Extract the contents of the boost-x.xx.x (x is a number) folder in the BOOST libaries archive you downloaded to $BOSS\lib\boost\ (you should have lib, include, boostrap.bat, etc. in this folder).
  4. Start a Command Prompt and cd to $BOSS\lib\boost and run bootstrap.bat
  5. In the Command Prompt, run bjam --toolset=msvc-9.0 --build-type=complete . This will take a while. Once that is done, you may close the Command Prompt window.

Preparing Curl

We are going to be building the curl requirements from source, since the Windows curl packages are often not up-to-date. We don't need to compile all of curl though, just the static library. To do so, follow these steps:

  1. Extract the contents of the curl-x.xx.x (x is a number) folder in the curl archive to $BOSS\lib\curl\ (you should have lib, include, README, etc. in this folder).
  2. Open $BOSS\lib\curl\lib\libcurl.vcproj in Visual C++ 2010. When the Conversion Wizard window appears, just click 'Finish' then 'Close'.
  3. Set the build type to Release. In Project->Properties->Configuration Properties->C/C++->Code Generation, set the Runtime Library to "Multi-threaded (/MT)".
  4. Build the libcurl project. This will create a libcurl.lib file in the $BOSS\lib\curl\lib\Release directory. An 'Unable to start' error message will pop up, ignore this and click 'OK'. Close the libcurl project.

Building BOSS

  1. Open $BOSS\code\boss-common\trunk\BOSS.sln in Visual C++ 2010.
  2. Set the build type to "Release" and run Build on the BOSS-Common project, which will give you BOSS.exe. Alternatively, run Debug->Build Solution to build all the projects in the solution.
  3. The generated exe(s) will be located in $BOSS\code\boss-common\trunk\bin\Release\.
